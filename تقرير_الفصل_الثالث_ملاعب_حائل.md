# الفصل الثالث: مشروع التدريب التعاوني (Coop Project)
## منصة «ملاعب حائل» لحجز وإدارة المرافق الرياضية البلدية

---

## 3.1 بيان المشكلة (Problem Statement)

تعاني مدينة حائل من حاجة متزايدة إلى تنظيم استخدام الملاعب والمرافق الرياضية التابعة للأمانة بطريقة إلكترونية حديثة. الاعتماد على الحجز التقليدي أو التواصل المباشر قد يؤدي إلى:

- صعوبة معرفة الأوقات المتاحة في كل حديقة وملعب.
- تعارض الحجوزات وتكرارها في نفس الوقت.
- ضعف متابعة الموظف للحجوزات والحدائق من واجهة واحدة.
- قلة الاستفادة من مبادرات **المدينة الذكية** والخدمات الرقمية البلدية.

لذلك ظهرت الحاجة إلى منصة إلكترونية تتيح للمواطنين حجز الملاعب بسهولة، وتمنح موظفي الأمانة لوحة تحكم لمتابعة الحجوزات وإدارة الحدائق.

---

## 3.2 نظرة عامة على المشروع (Project Overview)

**اسم المشروع:** منصة ملاعب حائل  
**الجهة المستفيدة:** أمانة منطقة حائل  
**نوع النظام:** تطبيق ويب (Web Application)

المنصة عبارة عن موقع إلكتروني لحجز وإدارة المرافق الرياضية البلدية في مدينة حائل، ويشمل:

1. **واجهة عامة للمستخدم** لعرض الحدائق والحجز عبر التاريخ والوقت.
2. **خريطة تفاعلية** توضح موقع الحديقة المختارة مع دبوس الموقع.
3. **لوحة تحكم للموظف** لعرض الحجوزات وإضافة حدائق جديدة وتحديث حالة الحجز.
4. **قاعدة بيانات SQLite** لحفظ الحدائق والحجوزات بشكل دائم.

### التقنيات المستخدمة
| الطبقة | التقنية |
|--------|---------|
| Backend | Python + Flask |
| Frontend | HTML + CSS + JavaScript |
| قاعدة البيانات | SQLite (`malaeb.db`) |
| الواجهة | RTL (عربي) بألوان مستوحاة من شعار أمانة حائل |

---

## 3.3 أهداف المشروع (Project Objectives)

1. توفير منصة إلكترونية سهلة لحجز الملاعب البلدية في حائل.
2. عرض الحدائق بأنواع الملاعب والصور والمواقع على الخريطة.
3. تمكين المستخدم من اختيار **التاريخ** و**الوقت** من جدول حجوزات واضح.
4. حفظ الحجوزات في قاعدة بيانات ومنع تعارض الأوقات المحجوزة.
5. توفير لوحة تحكم للموظف لـ:
   - عرض جميع الحجوزات.
   - تحديث حالة الحجز (مؤكد / مكتمل / ملغي).
   - إضافة حدائق جديدة.
6. دعم توجه المدينة الذكية وربط الخدمة بموقع أمانة منطقة حائل.
7. بناء نظام منظم وقابل للتطوير مستقبلاً (مثل الدفع الإلكتروني أو SMS حقيقي).

## 3.4 مراحل المشروع وحل المشكلة (Project Phases and Problem Solution)

### المرحلة 1: تحليل المشكلة والمتطلبات
- تحديد مستخدمي النظام: **زائر / مستخدم حجز** و **موظف الأمانة**.
- تحديد الصفحات الأساسية: الرئيسية، الحجز، دخول المستخدم، دخول الموظف، لوحة التحكم.

### المرحلة 2: تصميم الواجهات وقاعدة البيانات
- تصميم واجهة RTL بألوان الأخضر والرمادي المستوحاة من شعار الأمانة.
- تصميم جداول: `parks` و `bookings`.

### المرحلة 3: بناء الواجهة العامة
- الصفحة الرئيسية وتعريف المنصة والمدينة الذكية.
- صفحة الحجز: بطاقات الحدائق، الخريطة، الدبوس، جدول الأوقات، نموذج التأكيد.

### المرحلة 4: بناء لوحة تحكم الموظف
- تسجيل دخول الموظف.
- عرض الإحصائيات والحجوزات.
- إضافة حديقة وتحديث حالة الحجز.

### المرحلة 5: الربط بقاعدة البيانات والاختبار
- حفظ الحجوزات والحدائق في SQLite.
- اختبار تعارض الأوقات وتحديث الجدول بعد الحجز.

### كيف يحل المشروع المشكلة؟
| المشكلة | الحل في المنصة |
|---------|----------------|
| صعوبة معرفة الأوقات المتاحة | جدول حجوزات يعرض متاح / محجوز |
| تعارض الحجوزات | التحقق من قاعدة البيانات قبل التأكيد |
| ضعف متابعة الموظف | لوحة تحكم تعرض كل الحجوزات |
| صعوبة الوصول للموقع | خريطة Google Maps مع دبوس للحديقة |
| عدم وجود إدارة للحدائق | نموذج إضافة حديقة من لوحة الموظف |

## 3.5 عملية تطوير النظام (Project Development Process)

### منهجية التطوير
تم اتباع أسلوب تطوير تدريجي (Incremental):
1. إنشاء الهيكل الأساسي لـ Flask.
2. بناء الصفحات العامة.
3. تطوير فكرة الحجز (صور + تاريخ + جدول أوقات).
4. ربط الخريطة بالحديقة المختارة.
5. إضافة قاعدة البيانات ولوحة تحكم الموظف.
6. اختبار الوظائف وإصلاح الأخطاء.

### هيكل الملفات الرئيسي
```
app.py                 → المسارات والمنطق الرئيسي
database.py            → التعامل مع SQLite
malaeb.db              → ملف قاعدة البيانات
templates/             → صفحات HTML
static/css/style.css   → التنسيق
static/js/script.js    → تفاعل الحجز والخريطة وتسجيل الدخول
static/img/            → الشعار وصور الحدائق
```

### أهم المسارات (Routes)
| المسار | الوظيفة |
|--------|---------|
| `/` | الصفحة الرئيسية |
| `/booking` | صفحة حجز الملاعب |
| `/login` | دخول المستخدم (محاكاة OTP) |
| `/employee/login` | دخول الموظف |
| `/dashboard` | لوحة تحكم الموظف |
| `/api/bookings` | إنشاء حجز جديد |
| `/api/booked-slots` | جلب الأوقات المحجوزة حسب التاريخ |

### بيانات دخول الموظف (للتجربة)
- المستخدم: `employee`
- كلمة المرور: `1234`

## 3.7 نظرة عامة على واجهات النظام (System Interfaces Overview)

### أ) واجهة المستخدم العامة

#### 1) الصفحة الرئيسية `/`
- عنوان تعريف المنصة.
- نبذة عن الحجز والدفع الإلكتروني وإدارة الجداول.
- قسم المدينة الذكية في حائل.
- رابط إلى موقع أمانة منطقة حائل: https://amanathail.gov.sa/new_portal/

#### 2) صفحة الحجز `/booking`
- بطاقات الحدائق مع الصور والوصف وأنواع الملاعب.
- عند اختيار حديقة:
  - تنتقل الخريطة إلى موقعها.
  - يظهر دبوس باسم الحديقة.
- نموذج الحجز يحتوي على:
  - اسم المستخدم.
  - الحديقة.
  - نوع الملعب.
  - تاريخ الحجز.
  - الوقت المختار من الجدول.
  - عدد الساعات.
- جدول الأوقات من 16:00 إلى 22:00 بحالات: متاح / محجوز / اختيارك.
- بعد التأكيد تظهر رسالة نجاح ويُحفظ الحجز في قاعدة البيانات.

#### 3) صفحة تسجيل دخول المستخدم `/login`
- إدخال رقم الجوال.
- إرسال رمز تحقق (محاكاة).
- رمز التجربة: `1234`.

### ب) واجهة الموظف

#### 4) دخول الموظف `/employee/login`
- اسم مستخدم وكلمة مرور خاصة بالموظف.

#### 5) لوحة التحكم `/dashboard`
- إحصائيات: عدد الحدائق، إجمالي الحجوزات، الحجوزات المؤكدة.
- جدول الحجوزات مع إمكانية تغيير الحالة.
- قائمة الحدائق المسجلة.
- نموذج إضافة حديقة جديدة (الاسم، الموقع، الوصف، الإحداثيات).

### ج) العناصر المشتركة في كل الصفحات
- شعار أمانة منطقة حائل.
- اسم المنصة: ملاعب حائل.
- قائمة تنقل وروابط للصفحات.
- تذييل (Footer) يوضح هدف المنصة.

## نقاط مهمة للتقرير (خلاصة سريعة)

1. المشروع منصة ويب بلدية لحجز الملاعب في حائل.
2. يدعم مستخدمين اثنين: مستخدم حجز وموظف إدارة.
3. الحجز يتم عبر اختيار الحديقة والتاريخ والوقت من جدول واضح.
4. الخريطة ترتبط بالحديقة المختارة مع دبوس موقع.
5. قاعدة البيانات SQLite تخزن الحدائق والحجوزات.
6. لوحة التحكم تمكن الموظف من المتابعة والإضافة والتحديث.
7. التصميم عربي RTL بألوان شعار الأمانة.
8. النظام جاهز للتطوير لاحقاً (دفع إلكتروني، SMS، MySQL إن لزم).
